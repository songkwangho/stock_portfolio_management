# Skill: 한국 주식 앱 도메인 지식 가이드

이 문서는 Stock Analyzer 앱 개발에 필요한 한국 주식 시장 도메인 지식을 정리한다.
기능 구현 또는 UX 설계 시 참고한다.

---

## 1. 한국 주식 시장 기본 구조

### 거래소
- **KOSPI**: 삼성전자, 현대차 등 대형주. 종목코드 6자리 (예: 005930)
- **KOSDAQ**: 중소·기술주 중심. 종목코드 6자리 (예: 035720)
- **거래 시간**: 평일 09:00~15:30 KST. 공휴일 휴장.
- **가격 제한폭**: 전일 종가 대비 ±30%

### 주요 지표

| 지표 | 계산식 | 초보자 설명 | 낮으면 | 높으면 |
|------|--------|------------|--------|--------|
| PER | 주가 / EPS | 1년 이익의 몇 배에 사는가 | 저렴 가능 | 비쌀 수 있음 |
| PBR | 주가 / BPS | 장부가치 대비 배수 | 자산 저평가 | 자산 고평가 |
| ROE | 순이익 / 자기자본 × 100 | 투자 대비 수익률 | 효율 낮음 | 효율 높음 |
| EPS | 순이익 / 발행주식수 | 주당 순이익 | — | — |
| PEG | PER / EPS성장률 | 성장 보정 PER | 성장 대비 저평가 | 성장 대비 고평가 |

> PER 비교는 반드시 **같은 업종끼리**. IT와 금융을 PER로 비교하면 의미 없음.

### 적정 범위 (참고용, 업종별 편차 있음)
- PER: 10~20배 적정, 20배+ 고평가, 10 미만 저평가 가능
- PBR: 1 미만 자산 저평가, 3+ 고평가
- ROE: 5% 미만 낮음, 10~15% 양호, 15%+ 우량

## 2. 기술적 분석 지표

### RSI (상대강도지수)
- **계산**: 14일 평균 상승폭 / (상승폭 + 하락폭) × 100
- **범위**: 0~100
- **해석**: 70이상 과매수, 30이하 과매도, 30~50 하락 후 회복
- **한계**: 강한 추세에서 70 이상 지속 가능

### MACD
- **계산**: EMA(12) - EMA(26), 시그널 = EMA(9)
- **히스토그램**: MACD - 시그널
- **해석**: 히스토그램 양수·증가 = 상승 모멘텀
- **한계**: 후행 지표, 추세 전환 후 신호

### 볼린저밴드
- **계산**: 20일 이동평균 ± 2 표준편차
- **%B**: (현재가 - 하단) / (상단 - 하단) × 100
- **해석**: %B < 20 과매도, %B > 80 과매수

### 이동평균선 (SMA)
- **5일선**: 단기 (1주일), 가장 민감
- **20일선**: 중기 (1달), 중요 지지/저항
- **정배열**: 주가 > 5일선 > 20일선 → 상승 추세
- **역배열**: 주가 < 5일선 < 20일선 → 하락 추세

## 3. 수급 (투자자별 매매동향)

| 투자자 | 특징 | 초보자 설명 |
|--------|------|------------|
| 외국인 | 글로벌 기관, 정보력 높음 | 해외 큰손 |
| 기관 | 증권사, 보험, 펀드 | 국내 전문 투자 |
| 개인 | 일반 투자자 | 역발상 지표로 쓰이기도 |

이 앱의 수급 스코어: 외국인(1.2) + 기관(0.8), 10일 감쇠(decay=0.8).

## 4. HoldingOpinion 로직의 현실적 의미

| 판단 | 표시 | 시나리오 |
|------|------|---------|
| 손절(-7%) | 주의 필요 | 평단 대비 7%+ 하락 |
| 이중 이탈 | 주의 필요 | 5일선·20일선 모두 이탈 |
| 단기이탈+중기지지 | 관망 | 5일선 아래, 20일선 위 |
| 5일선 근접 | 추가 검토 | 5일선 ±1% 이내 |
| 정배열 | 보유 | 상승 추세 유지 |

**-7% 기준**: 알고리즘 참고값, 개인별 리스크 허용 범위에 따라 다름. Phase 4 백테스팅 후 최적화.

## 5. MarketOpinion 스코어링 한계

1. **백테스팅 미검증**: 7/4점 임계값은 임의 설정 (Phase 4에서 데이터 기반 최적화 예정)
2. **시장 환경 미반영**: 강세/약세장에 따라 같은 점수의 의미가 다름
3. **섹터 특수성**: 바이오는 PER 무의미, 금융은 PBR이 더 중요
4. **단기 지표 편중**: RSI/MACD는 단기 신호

### 섹터별 지표 특성

| 섹터 | PER 기준 | 주요 지표 |
|------|---------|---------|
| 기술/IT | 20~40배 정상 | ROE, 매출 성장률 |
| 금융/지주 | 5~15배 | PBR, 배당수익률 |
| 바이오 | 적자 많음 (무의미) | 파이프라인, 임상 단계 |
| 에너지/소재 | 10~20배 | 원자재 가격 |
| 자동차 | 8~15배 | 영업이익률 |

## 8. 포트폴리오 관리 UX

### 초보자 혼동 포인트

1. 평단가 vs 현재가 구분
2. 수익률 계산식: `(현재가 - 평단가) / 평단가 × 100`
3. 포트폴리오 비중: 전체 투자금 중 이 종목의 %
4. 평가금액: 현재가 × 보유 수량

### 폼 레이블 가이드

```
평단가 입력: "내가 산 평균 가격 (원)"
  힌트: "여러 번 나눠 샀다면 평균을 입력해요."

수량 입력: "보유 주식 수 (주)"
  힌트: "증권사 앱 → 보유 종목에서 확인할 수 있어요."
```
